IGB ERP 系統最終交付版本總覽 (V1.0.0)1. 系統核心與技術規格 (IGB ERP 系統與技術)文件日期: 2025年10月26日 上午8:24版本: V1.0.0IGB ERP 系統是整個企業資源規劃的核心基礎，負責處理所有的業務邏輯、數據儲存與安全管理。它是系統的「大腦與引擎」。項目描述架構模型採用微服務 (Microservices) 設計原則，確保高可用性與模組化擴展。數據層使用 Google Firestore 進行高彈性、高即時性的數據持久化儲存，支援跨區域同步。業務範圍涵蓋銷售管理、採購與供應鏈、庫存管理、以及財務會計等四大模組。核心目標確保數據的準確性、一致性與安全性，並提供穩定且低延遲的資料 API 服務。技術堆棧後端 API 服務基於 Node.js/TypeScript 運行，並部署於雲端環境。2. 核心成果展示 (IGB 核心增長系統互動式儀表板)成果交付日期: 2025年10月26日 上午8:41「互動式儀表板」是 V1.0.0 交付給終端用戶的關鍵介面，它是核心系統數據的「視覺化駕駛艙」。儀表板透過清晰、即時的方式呈現關鍵績效指標 (KPIs) 和操作數據。儀表板主要功能：即時數據監控: 透過四個 KPI 卡片（如總銷售額、待處理訂單），快速提供業務狀態概覽。趨勢分析: 內嵌 Chart.js 圖表，視覺化展示月度銷售趨勢，便於管理層決策。操作日誌: 提供最新的系統活動日誌，追蹤重要業務流程的變動。響應式設計: 介面採用 Tailwind CSS 構建，確保在桌面、平板和移動設備上皆能提供一致且優化的用戶體驗。3. 系統與儀表板的串接機制核心系統（後端）與儀表板（前端）之間的互動是透過標準化的 RESTful API 接口 和 即時數據訂閱 實現的。串接流程：數據請求 (Request): 當用戶登入儀表板時，前端會向後端 API 服務發出認證和數據請求（例如：GET /api/v1/dashboard/stats）。業務邏輯處理 (Core System): IGB ERP 系統（後端）執行必要的查詢、計算和安全驗證，從 Firestore 獲取最新的業務數據。數據傳輸 (Data Transfer): 核心系統將處理後的數據以標準 JSON 格式返回給儀表板。即時更新 (Real-time Link): 為了確保儀表板上的數據（如待處理訂單）能夠即時響應用戶操作，關鍵數據集合使用了 Firestore 的 onSnapshot 訂閱機制。這允許核心系統數據一有變動，儀表板即可無延遲地更新相應的圖表和卡片。總結:IGB ERP V1.0.0 成功地將穩健的後端業務引擎與現代化的互動式視覺介面結合，實現了數據的高效處理與即時展示，為企業的「核心增長」提供了可靠的技術支撐。
